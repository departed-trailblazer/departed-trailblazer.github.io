iOS11适配的一些坑点

iOS11 界面适配的坑点

一. tableView: heightForFooterInSection:和 tableView: heightForHeaderInSection: 代理方法不调用

原因:
iOS11 tableView 开启Self-Sizing 当你的组头视图或组尾视图为nil时，不调用上述两个方法，而是使用预估高度来设置组头和组尾高度。

解决方法:
实现 -tableView: viewForHeaderInSection:
-tableView: viewForFooterInSection:方法，返回一个UIView对象。或者将tableView组头和组尾的预估高度设为0:
tableView.estimatedSectionHeaderHeight = 0;
tableView.estimatedSectionFooterHeight = 0;

二. tableView setcontentOffSet 位置不准确

原因:
这里会取出所有Cell的高计算contentSize然后再滑动到offSet的位置 但是iOS11 默认开启了tableView的Self-Sizing 计算未展示过的cell的高度时是使用预估高度进行计算而不是实际高度。

解决方法:
tableView.estimatedRowHeight = 0;

三.iPhoneX StatusBar 高度适配

原因:
这个原因不说大家也都懂的，苹果大大要做全屏幕，但是显然技术上还不能把听筒和传感器的位置取代掉，所以了，就有了这个齐刘海的StatusBar。这里真的要吐槽一下这个齐刘海的手机外形，那一小块我觉得绝对可以逼死强迫症了，这让我觉得苹果在产品上已经不再追求完美了，开始降低对自己产品的苛刻要求了，小失落。。。

解决方案:
这还有啥说的，别偷懒，把项目中用到的20，64，44这些常量换成 [UIApplication sharedApplication].statusBarFrame.size.height、
self.navigationController.navigationBar.frame.size.height这些系统获取的高度，尽量使用autoLayout，可以把这些写成宏定义或者写进分类方法，看自己习惯了。
